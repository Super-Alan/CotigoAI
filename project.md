# Cogito AI 项目文档
### Cotigo AI
一款批判性思考的AI数智导师产品；包括 APP 端和 web 端，是一款现代设计风格，基于 AI Native 内核的产品； 首先我们实现 web 端的功能(要求兼容移动端)，后续再考虑是否扩展到 APP 端。

### 背景
这款产品首要目标时提升学生的批判性思考能力，通过提供一个互动的、有深度的思考环境，帮助学生理解复杂的问题、发现隐藏的假设、评估证据的可靠性，从而提升他们的批判性思维能力。

#### 升学对思维能力要求：
QS排名前10的高校（如香港大学、帝国理工学院、麻省理工、牛津、剑桥等）在面试中考察“批判性思维”，核心目标并非筛选“正确答案”，而是评估学生**拆解问题的逻辑、多角度论证的能力、质疑前提的意识，以及基于证据修正观点的灵活性**。这类问题的设计通常围绕5大核心分类，每个分类均对应高校对“未来学者/领导者”的思维能力诉求：


##### 一、现实议题的“多维归因与利弊权衡”类
这类问题聚焦当下全球或领域内的热点议题，要求学生跳出“非黑即白”的判断，从社会、经济、伦理、技术等多维度分析问题本质，同时权衡不同解决方案的成本与收益。  
**考察重点**：能否识别议题的复杂关联性，避免单一归因；能否在冲突的价值（如“效率vs公平”“创新vs风险”）中建立分析框架。  
**典型问题示例**：  
- 港大（社科/商科方向）：“香港推出‘高端人才通行证计划’后，部分行业出现‘人才内卷’，你认为该政策的核心矛盾是什么？若要优化，应优先考虑企业需求还是本地就业市场平衡？”  
- 帝国理工（理工科方向）：“欧盟拟2035年禁售燃油车，有人认为这能加速碳中和，也有人认为会导致传统汽车产业失业潮与供应链断裂。你如何评估这一政策的可行性？”  
**应对关键**：先明确议题的“核心矛盾点”（如政策目标与现实约束的冲突），再分维度拆解（如经济成本、社会影响、技术瓶颈），最后指出“权衡标准”（如短期阵痛与长期收益的优先级）。


##### 二、学术场景的“前提质疑与方法批判”类
这类问题紧密结合申请专业的学术逻辑，要求学生对学科内的经典理论、实验设计或研究结论进行“批判性审视”——而非被动接受知识，核心是考察“学术反思能力”。  
**考察重点**：能否识别理论/研究的“前提假设”（如样本局限性、变量控制缺陷）；能否区分“结论的适用边界”与“绝对化表述”。  
**典型问题示例**：  
- 帝国理工（生物医学方向）：“某研究通过小鼠实验证明‘某药物可抑制肿瘤生长’，为何该结论不能直接推广至人类临床？在设计人类临床试验时，需补充哪些控制条件？”  
- 牛津（经济学方向）：“‘看不见的手’理论假设‘市场参与者均为理性人’，但现实中存在‘非理性消费’（如冲动购物），这是否意味着该理论已失效？请说明你的判断依据。”  
**应对关键**：先拆解“原理论/研究的逻辑链”（前提→方法→结论），再针对每个环节提问（如“前提是否符合现实？”“方法是否遗漏关键变量？”），最后说明“修正方向”（如补充样本类型、调整假设条件）。


##### 三、逻辑论证的“谬误识别与证据评估”类
这类问题不依赖专业知识，而是聚焦“思维的严谨性”——要求学生识别论证中的逻辑漏洞（如偷换概念、以偏概全、因果倒置），或评估证据的“有效性与相关性”。  
**考察重点**：能否脱离“内容本身”，审视“论证过程的合理性”；能否区分“事实证据”与“主观观点”。  
**典型问题示例**：  
- 剑桥（人文社科通用）：“有人说‘过去10年全球平均气温上升，同时全球风电装机量增长，因此风电加剧了全球变暖’，这个论证存在什么问题？你需要补充哪些证据才能验证两者的关系？”  
- 港大（法学/新闻方向）：“某媒体报道‘某地区青少年犯罪率上升，而该地区近3年新增3所网吧，因此网吧是青少年犯罪的主因’。这种报道逻辑是否严谨？为何？”  
**应对关键**：先定位“论证结构”（论据→论点），再指出“逻辑谬误类型”（如本例中的“因果倒置”“第三变量遗漏”，如“犯罪率上升可能与社会失业率相关，而非网吧”），最后说明“有效证据的标准”（如控制其他变量后的对比数据）。


##### 四、自我认知的“观点迭代与反思”类
这类问题以学生的个人经历、观点或选择为切入点，考察“元认知能力”——即能否审视自己过去的判断，承认认知局限，并基于新信息调整观点，而非固执己见。  
**考察重点**：能否跳出“自我辩护”，客观分析自己的思维盲区；能否体现“成长型思维”（从错误/挑战中修正认知）。  
**典型问题示例**：  
- 麻省理工（工程方向）：“你在高中的科创项目中，曾认为‘某设计方案最优’，但最终实验结果未达预期。现在回头看，你当时的判断忽略了哪些关键因素？如果重新设计，你会如何调整思路？”  
- 港大（商科方向）：“你之前提到‘更倾向于进入金融行业，因为薪资高’，但近年来金融业裁员潮频发，且部分领域对社会责任的争议较大。这一信息是否会改变你的职业选择？请说明理由。”  
**应对关键**：不回避“过去的不足”，而是将其转化为“反思的切入点”，重点说明“如何通过新信息/新经验修正认知”（如“当时忽略了材料的耐温性，现在会先做多环境下的测试”）。


##### 五、跨领域的“关联与迁移”类
这类问题要求学生打破学科或场景的边界，将A领域的逻辑/方法迁移到B领域，考察“系统性思维”——即能否识别不同问题背后的共性逻辑，并用跨学科视角解决复杂问题。  
**考察重点**：能否跳出“单一学科思维”，建立“跨界关联”；能否用简洁的逻辑解释复杂的跨界应用。  
**典型问题示例**：  
- 帝国理工（计算机方向）：“‘囚徒困境’是经济学中的经典模型，你认为这一模型能否用于解释人工智能算法的‘伦理冲突’（如‘算法优化效率’与‘保护用户隐私’的矛盾）？如何应用？”  
- 牛津（环境科学方向）：“‘供应链管理’的核心是‘优化资源流动效率’，这一思路能否用于设计‘垃圾分类回收体系’？请具体说明可能的迁移点。”  
**应对关键**：先提炼A领域的“核心逻辑”（如囚徒困境的“个体理性与集体理性冲突”），再找到B领域的“相似矛盾点”（如AI算法中“企业利益与用户权益的冲突”），最后说明“迁移后的应用方式”。


##### 核心总结：这些问题的“共性逻辑”
无论属于哪一类，QS前10高校的批判性思维问题都遵循一个底层逻辑：**“不考‘你知道什么’，而考‘你如何思考’”**。面试者无需追求“完美答案”，但需展现三个关键特质：  
1. **拆解问题的结构化能力**：不混乱发散，而是按“维度/步骤”有序分析；  
2. **质疑的建设性**：不盲目否定，而是“基于证据质疑，基于逻辑修正”；  
3. **视角的开放性**：不固守单一立场，而是“承认多角度的合理性，并说明自己的权衡标准”。  

本质上，这些问题是高校对“未来学术/职业场景”的模拟——未来面对的复杂问题（如气候危机、技术伦理）均无标准答案，唯有具备批判性思维的人，才能持续探索解决方案。


### **核心设计哲学：从“答案提供者”到“思维脚手架”**

传统AI（如简单的搜索或问答）倾向于直接给出“答案”，这会扼杀用户的独立思考。我们的核心理念是反其道而行之：**Cogito AI 不直接提供答案，而是扮演一个“苏格拉底式”的数字导师，通过不断追问、提供多元视角、解构论证，来激发和锻炼用户自己的思维深度和广度。**


### **最核心的MVP功能清单 (Minimum Viable Product Feature List)**

MVP的目标是，用最少的功能，验证核心价值。以下三个功能足以构成一个强大且令人印象深刻的MVP。

#### **功能一：苏格拉底对话机器人 (Socratic Dialogue Module)**

这是APP的绝对核心，它实现了“苏格拉底式”的对话模式，为用户提供一个互动的、有深度的思考环境；所有对话历史需要进行持久化存储，可参考同类一流 AI 大模型产品的设计如 Open AI chat box。

* **用户流程:**
    1.  用户输入一个议题、观点或一篇文章的链接。例如：“我认为人工智能最终会取代所有人类工作。”
    2.  Cogito AI **不会**直接反驳或赞同，而是开始提问：
        * “你说的‘所有工作’具体指哪些类型？是否包括创造性或情感类的工作？” (澄清概念)
        * “你得出这个结论是基于哪些事实或证据？” (追溯证据)
        * “这个观点背后，可能隐藏着哪些你没有意识到的假设？” (挖掘深层假设)
        * “有没有可能存在一个完全相反的、但同样有道理的观点？它会是什么样的？” (引导换位思考)
    3.  整个对话过程，AI会不断引导用户深化思考，而不是灌输知识。

* **技术实现:**
    * 后端通过精心设计的系统提示词（System Prompt）来设定AI的角色。例如：“你是一名苏格拉底式的哲学导师，你的唯一目标是通过提问来挑战和启发用户，你绝不能直接给出答案或自己的观点。你的提问应该遵循[澄清概念 -> 检验证据 -> 挖掘假设 -> 探索其他视角]的逻辑链条。”

#### **功能二：论点解构器 (Argument Deconstructor)**

这个功能旨在将抽象的思维过程可视化。

* **用户流程:**
    1.  用户粘贴一段文字（例如新闻评论、论文摘要）。
    2.  点击“解构”按钮。
    3.  Cogito AI 将文本进行结构化分析，并以卡片或思维导图的形式呈现：
        * **核心主张 (Main Claim):** 这段话最想表达的观点是什么？
        * **论据 (Evidence):** 作者用了哪些事实、数据或例子来支持主张？
        * **隐藏假设 (Underlying Assumptions):** 要让这个论证成立，作者默认了哪些前提条件是真的？
        * **潜在逻辑谬误 (Potential Fallacies):** AI会尝试识别常见的逻辑谬误，如“稻草人谬误”、“人身攻击”、“滑坡谬误”等，并给出解释。

* **技术实现:**
    * 这需要更复杂的Prompt。后端会将用户文本嵌入一个指令模板中，要求LLM分步进行抽取和识别，并以JSON格式返回结果，方便前端进行可视化展示。

#### **功能三：多棱镜视角 (Perspective Prism)**

这个功能旨在打破信息茧房，培养同理心和全局观。

* **用户流程:**
    1.  用户输入一个具有争议性的话题，例如“是否应该对富人征收更高的税？”
    2.  用户可以选择希望AI扮演的“角色”，例如：一位自由市场经济学家、一位社会主义活动家、一位中产阶级纳税人、一位政府财政部长。
    3.  Cogito AI 会从用户选择的每个角色的立场出发，生成一段逻辑严密、论据充分的观点陈述。
    4.  用户可以“追问”任何一个角色，与其进行深入对话。

* **技术实现:**
    * 通过动态改变Prompt中的角色设定（"You are a..."）来实现。后端管理不同的角色“性格卡”，并生成相应的观点。

### 技术栈选型 (Technology Stack)
#### 核心架构
    全栈框架 : Next.js 14 + TypeScript   开发模式 : AI Native + 现代化全栈

#### 前端技术栈
##### 核心框架
- Next.js 14 (App Router + Server Components)
- React 18 (并发特性 + Suspense)
- TypeScript (严格类型检查)
##### UI & 样式
- Tailwind CSS (原子化CSS)
- shadcn/ui (现代组件库)
- Framer Motion (动画库，适合思维导图)
- Lucide React (图标库)
##### 状态管理
- Zustand (轻量状态管理)
- React Query/TanStack Query (服务端状态)
- React Hook Form (表单管理)
##### 数据可视化
- D3.js (论点解构的思维导图)
- Recharts (数据图表)
- React Flow (节点关系图)
#### ⚡ 后端技术栈
##### API 层
- Next.js API Routes (RESTful API)
##### 数据库层
- Prisma ORM (类型安全的数据库操作)
- PostgreSQL (主数据库)
- Redis (缓存)

##### 认证层
- NextAuth.js (OAuth2 + JWT)
- 密码哈希 (bcrypt)
- 会话管理 (NextAuth.js)
